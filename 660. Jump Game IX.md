# Problem Statement

You are given an integer array `nums`.

From any index `i`, you can jump to another index `j` under the following rules:

- Jump to index `j` where `j > i` is allowed only if `nums[j] < nums[i]`
- Jump to index `j` where `j < i` is allowed only if `nums[j] > nums[i]`

### 🎯 Goal

For each index `i`, find the **maximum value in `nums`** that can be reached by following any sequence of valid jumps starting at `i`.

Return an array `ans` where:


ans[i] = maximum reachable value starting from index i


---

## 🧪 Examples

### Example 1
Input:  
nums = [2,1,3]  

Output:  
[2,2,3]  

---

### Example 2
Input:  
nums = [2,3,1]  

Output:  
[3,3,3]  

---

## 📌 Constraints

- 1 ≤ nums.length ≤ 10⁵  
- 1 ≤ nums[i] ≤ 10⁹  

---

## 🧠 Approach

### 💡 Key Insight

This problem can be seen as a **graph problem**:

- Each index = node  
- Valid jumps = directed edges  

We need to find the **maximum value reachable from each node**.

---

### ⚡ Optimization Trick

Instead of checking all `O(n²)` jumps:

👉 Use **Monotonic Stack** to find:
- Next smaller element on the right  
- Next greater element on the left  

These define valid transitions efficiently.

---

### 🔁 Strategy

1. Build graph using monotonic stacks:
   - Forward edges (next smaller)
   - Backward edges (next greater)

2. Use **DFS + Memoization**:
   - Compute max reachable value from each index  
   - Cache results to avoid recomputation
