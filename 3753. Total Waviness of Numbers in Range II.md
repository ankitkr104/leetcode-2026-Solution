## 📘 Problem Statement

You are given two integers:

```text
num1, num2
```

representing the inclusive range:

```text
[num1, num2]
```

The **waviness** of a number is defined as the total number of **peaks** and **valleys** in its decimal representation.

### Peak

A digit is a peak if:

```text
digit > left neighbor
AND
digit > right neighbor
```

### Valley

A digit is a valley if:

```text
digit < left neighbor
AND
digit < right neighbor
```

### Notes

- The first and last digits can never be peaks or valleys.
- Any number with fewer than 3 digits has waviness `0`.

Return the total sum of waviness for all numbers in the range:

```text
[num1, num2]
```

---

## 🧪 Examples

### Example 1

```text
Input:
num1 = 120
num2 = 130

Output:
3
```

### Explanation

```text
120 → 1
121 → 1
130 → 1

Total = 3
```

---

### Example 2

```text
Input:
num1 = 198
num2 = 202

Output:
3
```

### Explanation

```text
198 → Peak at 9 → 1
201 → Valley at 0 → 1
202 → Valley at 0 → 1

Total = 3
```

---

### Example 3

```text
Input:
num1 = 4848
num2 = 4848

Output:
2
```

### Explanation

```text
4848

8 is a peak
4 is a valley

Waviness = 2
```

---

## 📌 Constraints

```text
1 <= num1 <= num2 <= 10^15
```

---

## 🧠 Approach

### 💡 Key Observation

The range can be as large as:

```text
10^15
```

which makes brute force impossible.

Instead, we use:

```text
Digit Dynamic Programming (Digit DP)
```

to compute:

```text
waviness(0...X)
```

and then use:

```text
Answer =
waviness(0...num2)
-
waviness(0...num1-1)
```
