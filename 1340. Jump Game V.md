# Problem Statement

You are given:
- an integer array `arr`
- an integer `d`

From index `i`, you can jump to:

```text
i + x
i - x
```

where:
```text
1 <= x <= d
```

But you can only jump from `i` to `j` if:

```text
arr[i] > arr[j]
```

AND every element between `i` and `j` is also smaller than `arr[i]`.

Return the **maximum number of indices** you can visit.

You may start from **any index**.

---

## 🧪 Examples

### Example 1
```text
Input:
arr = [6,4,14,6,8,13,9,7,10,6,12]
d = 2

Output:
4
```

Explanation:
```text
10 → 8 → 6 → 7
```

---

### Example 2
```text
Input:
arr = [3,3,3,3,3]
d = 3

Output:
1
```

---

### Example 3
```text
Input:
arr = [7,6,5,4,3,2,1]
d = 1

Output:
7
```

---

## 📌 Constraints

```text
1 <= arr.length <= 1000
1 <= arr[i] <= 10^5
1 <= d <= arr.length
```

---

## 🧠 Approach

### 💡 Key Insight

This is a:
```text
DFS + Memoization
```

problem.

For every index:
- try jumping left
- try jumping right
- compute maximum reachable path length

Store results using DP.
