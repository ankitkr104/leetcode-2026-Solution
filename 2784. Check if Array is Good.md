# Problem Statement

An array is considered **good** if it is a permutation of:

```text
base[n] = [1, 2, ..., n - 1, n, n]
```

Meaning:
- Numbers from `1` to `n-1` appear exactly once
- Number `n` appears exactly twice

Return `true` if the given array is good, otherwise return `false`.

---

## 🧪 Examples

### Example 1
```text
Input:
nums = [2,1,3]

Output:
false
```

Explanation:
```text
max(nums) = 3

base[3] = [1,2,3,3]
Length mismatch → false
```

---

### Example 2
```text
Input:
nums = [1,3,3,2]

Output:
true
```

Explanation:
```text
nums is a permutation of [1,2,3,3]
```

---

### Example 3
```text
Input:
nums = [1,1]

Output:
true
```

---

### Example 4
```text
Input:
nums = [3,4,4,1,2,1]

Output:
false
```

---

## 📌 Constraints

```text
1 <= nums.length <= 100
1 <= nums[i] <= 200
```

---

## 🧠 Approach

### 💡 Key Idea

If:
```text
n = max(nums)
```

Then a good array must:
- have length `n + 1`
- contain:
```text
[1,2,3,...,n-1,n,n]
```

---

## 🔁 Steps

1. Find maximum element `n`
2. Build expected array:
```text
[1,2,...,n,n]
```
3. Sort both arrays
4. Compare

---

# Solution (Python)

```python
class Solution:
    def isGood(self, nums):
        n = max(nums)

        expected = list(range(1, n + 1))
        expected.append(n)

        return sorted(nums) == expected
```
