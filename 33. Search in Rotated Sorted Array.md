# Problem Statement

You are given a sorted array `nums` that may be rotated at some pivot.

Example:
```text
[0,1,2,4,5,6,7]
```

might become:
```text
[4,5,6,7,0,1,2]
```

Given:
- rotated array `nums`
- integer `target`

Return:
- index of `target` if found
- otherwise `-1`

⚠️ Required Time Complexity:
```text
O(log n)
```

---

## 🧪 Examples

### Example 1
```text
Input:
nums = [4,5,6,7,0,1,2]
target = 0

Output:
4
```

---

### Example 2
```text
Input:
nums = [4,5,6,7,0,1,2]
target = 3

Output:
-1
```

---

### Example 3
```text
Input:
nums = [1]
target = 0

Output:
-1
```

---

## 📌 Constraints

```text
1 <= nums.length <= 5000
-10^4 <= nums[i] <= 10^4
All values are unique
nums is sorted and possibly rotated
-10^4 <= target <= 10^4
```

---

## 🧠 Approach

### 💡 Key Insight

Even after rotation:
```text
at least one half remains sorted
```

Use Binary Search to determine:
- which half is sorted
- whether target lies there

---

## 🔁 Binary Search Logic

At every step:

### Check left sorted half
If:
```text
nums[left] <= nums[mid]
```

then left half is sorted.

Now:
- if target lies inside left half
  → search left
- otherwise
  → search right

---

### Otherwise right half is sorted

Check if target lies there.

---

# Solution (Python)

```python
class Solution:
    def search(self, nums, target):

        left = 0
        right = len(nums) - 1

        while left <= right:

            mid = (left + right) // 2

            if nums[mid] == target:
                return mid

            # Left half sorted
            if nums[left] <= nums[mid]:

                if nums[left] <= target < nums[mid]:
                    right = mid - 1
                else:
                    left = mid + 1

            # Right half sorted
            else:

                if nums[mid] < target <= nums[right]:
                    left = mid + 1
                else:
                    right = mid - 1

        return -1
```
