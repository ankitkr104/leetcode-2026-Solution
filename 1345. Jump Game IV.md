# Problem Statement

You are given an integer array `arr`.

You start at index `0`.

In one step, you can jump from index `i` to:

```text
i + 1   (if within bounds)
i - 1   (if within bounds)
j where arr[i] == arr[j] and i != j
```

Return the **minimum number of steps** required to reach the last index.

⚠️ You cannot jump outside the array.

---

## 🧪 Examples

### Example 1
```text
Input:
arr = [100,-23,-23,404,100,23,23,23,3,404]

Output:
3
```

Explanation:
```text
0 → 4 → 3 → 9
```

---

### Example 2
```text
Input:
arr = [7]

Output:
0
```

Explanation:
```text
Already at last index
```

---

### Example 3
```text
Input:
arr = [7,6,9,6,9,6,9,7]

Output:
1
```

Explanation:
```text
0 → 7
```

---

## 📌 Constraints

```text
1 <= arr.length <= 5 * 10^4
-10^8 <= arr[i] <= 10^8
```

---

## 🧠 Approach

### 💡 Key Insight

This is a **shortest path problem**.

Each index behaves like a graph node.

Possible edges:
- left neighbor
- right neighbor
- all indices with same value

Use:
```text
BFS
```

because every jump costs exactly `1`.

---

## 🔥 Optimization Trick

If we repeatedly visit all same-value indices,
time complexity becomes huge.

Example:
```text
[7,7,7,7,7,7...]
```

To avoid TLE:
✅ After processing indices of one value,
remove them from the hashmap.

---

## 🔁 BFS Strategy

1. Store:
```text
value → list of indices
```

2. BFS from index `0`

3. Explore:
- `i - 1`
- `i + 1`
- all same-value indices

4. Use visited set

5. Delete processed value group
