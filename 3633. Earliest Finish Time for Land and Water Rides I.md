# Problem Statement

A tourist must take:

```text
Exactly one Land Ride
Exactly one Water Ride
```

in any order.

### Land Rides

```text
landStartTime[i]  = earliest boarding time
landDuration[i]   = ride duration
```

### Water Rides

```text
waterStartTime[j] = earliest boarding time
waterDuration[j]  = ride duration
```

Rules:

- A ride can start at its opening time or later.
- After finishing the first ride, the tourist immediately boards the second ride if it is open, otherwise waits.
- Return the earliest possible time to finish both rides.

---

## 🧪 Examples

### Example 1

```text
Input:
landStartTime = [2,8]
landDuration  = [4,1]

waterStartTime = [6]
waterDuration  = [3]

Output:
9
```

Explanation:

```text
Land(2→6)
Water(6→9)

Answer = 9
```

---

### Example 2

```text
Input:
landStartTime = [5]
landDuration  = [3]

waterStartTime = [1]
waterDuration  = [10]

Output:
14
```

---

## 📌 Constraints

```text
1 <= n,m <= 100

1 <= landStartTime[i], landDuration[i] <= 1000

1 <= waterStartTime[j], waterDuration[j] <= 1000
```

---

## 🧠 Approach

### 💡 Observation

For every pair:

```text
Land Ride i
Water Ride j
```

we try both possible orders:

### Option 1

```text
Land → Water
```

### Option 2

```text
Water → Land
```

Compute finish time for both and keep the minimum.

Since:

```text
n,m <= 100
```

Brute force over all pairs is sufficient.
