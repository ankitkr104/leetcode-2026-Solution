# Problem Statement

Suppose an array sorted in ascending order is rotated between `1` and `n` times.

Example:
```text
[0,1,4,4,5,6,7]
```

might become:
```text
[4,5,6,7,0,1,4]
```

Given the rotated sorted array `nums` that **may contain duplicates**, return the minimum element.

You should minimize the number of operations as much as possible.

---

## 🧪 Examples

### Example 1
```text
Input:
nums = [1,3,5]

Output:
1
```

---

### Example 2
```text
Input:
nums = [2,2,2,0,1]

Output:
0
```

---

## 📌 Constraints

```text
n == nums.length
1 <= n <= 5000
-5000 <= nums[i] <= 5000
nums is sorted and rotated between 1 and n times
```

---

## 🧠 Approach

### 💡 Key Insight

This problem is similar to:
```text
153. Find Minimum in Rotated Sorted Array
```

BUT:
```text
duplicates are allowed
```

Duplicates create ambiguity in binary search.

---

## 🔁 Binary Search Logic

Compare:
```text
nums[mid]
```

with:
```text
nums[right]
```

---

### Case 1
If:
```text
nums[mid] > nums[right]
```

Minimum lies in:
```text
right half
```

Move:
```text
left = mid + 1
```

---

### Case 2
If:
```text
nums[mid] < nums[right]
```

Minimum lies in:
```text
left half including mid
```

Move:
```text
right = mid
```

---

### Case 3 (Duplicates)
If:
```text
nums[mid] == nums[right]
```

We cannot determine the side.

Safely reduce search space:
```text
right -= 1
```
