# Problem Statement

You are given an integer array `nums` of length `n`.

You start at index `0`, and your goal is to reach index `n - 1`.

From any index `i`, you may perform one of the following operations:

### 🔹 Adjacent Step
- Jump to index `i + 1` or `i - 1`
- Only if the index is within bounds

### 🔹 Prime Teleportation
If `nums[i]` is a prime number `p`, you may instantly jump to any index `j != i` such that:


nums[j] % p == 0


---

## 🎯 Goal

Return the **minimum number of jumps** required to reach index `n - 1`.

---

## 🧪 Examples

### Example 1
Input:
nums = [1,2,4,6]

Output:
2

Explanation:
0 → 1 (adjacent step)  
1 → 3 (teleport using prime 2)  

---

### Example 2
Input:
nums = [2,3,4,7,9]

Output:
2

Explanation:
0 → 1 (adjacent step)  
1 → 4 (teleport using prime 3)  

---

### Example 3
Input:
nums = [4,6,5,8]

Output:
3

Explanation:
0 → 1 → 2 → 3  

No useful teleportation exists.

---

## 📌 Constraints

- 1 ≤ n ≤ 10⁵  
- 1 ≤ nums[i] ≤ 10⁶  

---

## 🧠 Approach

### 💡 Key Idea

This is a **shortest path problem** on an implicit graph.

Since every jump costs `1`,
👉 use **Breadth First Search (BFS)**.

---

## 🔁 Graph Connections

- Jump to all indices where:
