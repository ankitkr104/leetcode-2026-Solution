# Problem Statement

You are given a string `moves` of length `n` consisting of characters:
- `'L'` → move left
- `'R'` → move right
- `'_'` → can choose either left or right

You start from position `0` on a number line.

---

### Task

Return the **maximum distance from the origin** that you can reach after performing all moves.

---

## 🔍 Examples

### Example 1


Input: moves = "L_RL__R"
Output: 3


**Explanation:**
Choose moves to maximize distance → `"LLRLLLR"` → reach `-3`

---

### Example 2


Input: moves = "R__LL"
Output: 5


---

### Example 3


Input: moves = "_______"
Output: 7


---

## ⚙️ Constraints

- `1 <= moves.length <= 50`
- Only `'L'`, `'R'`, `'_'` are present

---
## 💡 Solution

### 🔹 Approach: Greedy Counting

- To maximize distance:
  - Convert all `'_'` into **one direction**
- Count:
  - Number of `'L'`
  - Number of `'R'`
  - Number of `'_'`

👉 Two possibilities:
- Go maximum **right**
- Go maximum **left**

Take the maximum of both.

---

### 💻 Code

```python
class Solution:
    def furthestDistanceFromOrigin(self, moves):
        left = moves.count('L')
        right = moves.count('R')
        blank = moves.count('_')
        
        # Max distance possible
        return abs(left - right) + blank
