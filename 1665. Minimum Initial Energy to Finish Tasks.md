# Problem Statement

You are given an array `tasks` where:

```text
tasks[i] = [actual_i, minimum_i]
```

- `actual_i` = energy spent to complete the task
- `minimum_i` = minimum energy required to start the task

You can complete tasks in **any order**.

Return the **minimum initial energy** required to finish all tasks.

---

## 🧪 Examples

### Example 1
```text
Input:
tasks = [[1,2],[2,4],[4,8]]

Output:
8
```

Explanation:
```text
Start with 8 energy

Do [4,8] → energy = 4
Do [2,4] → energy = 2
Do [1,2] → energy = 1
```

---

### Example 2
```text
Input:
tasks = [[1,3],[2,4],[10,11],[10,12],[8,9]]

Output:
32
```

---

### Example 3
```text
Input:
tasks = [[1,7],[2,8],[3,9],[4,10],[5,11],[6,12]]

Output:
27
```

---

## 📌 Constraints

```text
1 <= tasks.length <= 10^5
1 <= actual_i <= minimum_i <= 10^4
```

---

## 🧠 Approach

### 💡 Greedy Insight

For a task:

```text
[actual, minimum]
```

The extra buffer needed is:

```text
minimum - actual
```

Tasks with larger:

```text
(minimum - actual)
```

should be done earlier.

---

## 🔁 Sorting Strategy

Sort tasks by:

```text
(minimum - actual) descending
```

Why?

Because tasks needing larger reserve energy are harder to start later.

---

## ⚡ Greedy Simulation

Maintain:
- `energy` → current available energy
- `ans` → minimum initial energy needed

For each task:
1. If current energy < required minimum:
   - Increase initial energy
2. Complete task
3. Reduce energy by actual cost
